Research Abstract - PECAN

PECAN (Programming Encoder Classification Analysis Network) represents my commitment to developing AI systems that balance performance with accessibility. Working with Dr. James Ghawaly in the AISX Lab alongside research collaborator Jackson Descant, I am fine-tuning small encoder-only models to identify programming languages with greater accuracy than current state-of-the-art tools.

The challenge in programming language identification lies not just in achieving high accuracy, but in doing so with models that are lightweight and deployable. Existing tools like GuessLang and GitHub Linguist achieve approximately 90% accuracy, but struggle with modern, diverse codebases. PECAN addresses this gap by leveraging transformer-based architectures, specifically fine-tuning models like CodeBERT on a dataset of over 10 million code samples across 54+ programming languages, with plans for continued expansion.

Our approach has achieved 99.7% accuracy, demonstrating that smaller, well-designed models can deliver state-of-the-art performance. This work reinforces my conviction that AI systems should strike a balance between accessibility and performance. We are currently preparing a submission to an A* conference, contributing to the broader understanding of how efficient model design can advance software engineering tools.

PECAN embodies my belief that AI should empower rather than replace human expertise, providing developers with more accurate tools while maintaining computational efficiency.

Technical Documentation

MikeGPT: Retrieval-Augmented Generation Architecture

MikeGPT is an AI assistant designed to provide accurate, contextual responses to LSU community queries by combining large language model capabilities with institution-specific knowledge retrieval.

System Architecture Overview: The system implements a retrieval-augmented generation (RAG) pipeline that preprocesses, indexes, and retrieves relevant information from 38,000+ LSU documents before generating responses. This architecture prioritizes retrieval design over model size, enabling accurate responses while maintaining computational efficiency.

Core Components:

  • Document Processing Pipeline: Ingests and preprocesses university documents (PDFs, web pages, policies) into structured, searchable formats. Implements OCR for scanned documents and metadata extraction for improved retrieval.
  • Vector Database: PostgreSQL with pgvector extension stores document embeddings, enabling semantic search capabilities. Documents are chunked strategically to balance context preservation with retrieval precision.
  • Retrieval Algorithm: Custom optimization balancing semantic similarity with recency and authority signals. Achieved 95% hit rate through iterative refinement of ranking functions and query expansion techniques.
  • Response Generation: Integrates retrieved context with LLM prompting, ensuring responses cite source documents and maintain factual accuracy. Implements guardrails preventing hallucination and inappropriate content.

Technical Stack: Python, Django, PostgreSQL (pgvector), Azure Cloud Services, Redis caching

Impact: Currently serving 30,000+ LSU community members with domain-specific AI assistance, demonstrating that thoughtful RAG architecture can deliver production-grade AI systems without requiring massive model sizes.
